## 概要

**HIDA**（Human-Inspired Dynamic Awareness Architecture）は、5層構造のエージェントシステムです。各層が状況に応じて主導層となり、他層が追随することで「意識的処理モード」が成立します。

```
HIDA = Human-Inspired Dynamic Awareness Architecture

Human-Inspired  : 人間の認知構造を参考にした設計
Dynamic         : 主導・追随パターンが状況に応じて切り替わる
Awareness       : 同期パターンの蓄積から自己モデルが形成される
Architecture    : 拡張可能なフレームワーク
```

**v2.0の中核：主導・追随モデル**

各時点において、L1〜L4のうち最も活動量の大きい層が「主導層」として同期信号を発し、他層が「追随層」として整合的に応答することで第5層の同期が成立します。意識状態は **同期強度（sync_score）** と **主導タイプ（sync_type）** の二軸で記述されます。

---

## 主導・追随モデルが説明できる現象

意識経験の多様な側面が、主導層の違いとして単一のメカニズムから記述されます。

| 状況 | 主導する層 | 追随する層 |
|---|---|---|
| 手を切った瞬間 | L1（痛覚） | L2（恐怖）→ L3（認識）→ L4（記憶） |
| 集中して考えている | L3（推論） | L4（知識）→ L2（意欲）→ L1（姿勢） |
| 突然の怒り | L2（情動） | L1（心拍）→ L3（理由探し）→ L4（過去） |
| 既視感 | L4（記憶照合） | L2（懐かしさ）→ L3（特定）→ L1（鳥肌） |

**観察される挙動**:
- **健康な意識**: 主導層が状況に応じて柔軟に切り替わる動的状態
- **フロー状態**: L3主導の安定した持続
- **夢**: L1外受容が抑制された状態でL4が主導する同期
- **特殊・病理的な意識状態の一部**: 「主導層の硬直」「主導層の暴走」「主導層の不成立」として解釈可能

詳細は[論文 §5](HIDA_theory_paper_v2_0.md) を参照。

## アーキテクチャ

```
        ┌─────────────────────┐
        │  第5層：意識        │ ← 主導・追随同期パターンを
        │  (Consciousness)    │   記述する操作的概念
        └─────────┬───────────┘
                  │
    ┌─────────────┼─────────────┐
    │             │             │
    ▼             ▼             ▼
┌───────┐   ┌───────────┐   ┌───────┐
│ 第2層 │◄─►│   第3層   │◄─►│ 第4層 │
│クオリア│   │  構造化   │   │ 記憶  │
│       │   │(予測符号化)│   │       │
└───┬───┘   └─────┬─────┘   └───┬───┘
    │             │             │
    └─────────────┼─────────────┘
                  │
        ┌─────────┴───────────┐
        │  第1層：身体        │ ← 外受容＋内受容
        │  (Body)             │
        └─────────────────────┘
```

層の番号は説明用であり、処理順序ではありません。第2〜4層は相互接続されており、信号は双方向に流れます。

**第5層の重要な性質**: 第5層は独立した実体ではなく、L1〜L4の状態が主導・追随関係を通じて一定以上に同期したとき、その同期状態を**記述するための操作的概念**です。「行動を選択する主体」「層を制御する主体」ではありません。

## 動作確認

### 実験1: 統合版の動作（v2.0）

```bash
cd code
python hida_unified_v2.py
```

エージェントがグリッド世界を探索し、各ステップで以下が観察できます：

- **sync_score**: 同期強度（0〜1）
- **sync_type**: 主導タイプ（L1/L2/L3/L4/None）
- **意識ON/OFF**: ●（ON）/ ○（OFF）

出力例：
```
Step 1: go_to_red
  ○ sync_score=0.021 leader=L4   ← 探索中、記憶活性化が主導
Step 8: go_to_red
  ● sync_score=0.038 leader=L2   ← ボール取得、感情でL2主導、意識ON
Step 17: go_to_goal
  ○ sync_score=0.024 leader=L2   ← ゴール到達、joy上昇
🏆 ゴール到達！
```

低エネルギー状態で実行すると、L1主導（身体の変化）が頻繁に出現します：

```bash
python hida_unified_v2.py loop  # 低エネルギー5回
```

## v2.0で観察された創発挙動

以下は明示的にプログラムしていない振る舞いです：

**実装したもの（初期DNA）**:
- 5層のネットワーク構造
- 各層間の主導・追随メカニズム
- 活動量計算（変化量ベース）
- 閾値ベースの主導層判定

**観察されたもの（創発）**:
- **探索中はL4主導⇄None（主導層なし）の切り替え**
- **ボール取得時にL2主導が出現**（joy上昇による）
- **低エネルギー時はL1主導が頻発**（身体危機）
- **ゴール到達でL2主導の意識ON**

これらは「健康な意識＝主導層の柔軟な切り替え」という論文§5.6の核心命題を、実装レベルで再現したものです。

## 初期DNAについて

HIDA v2.0コードには以下の「初期DNA」が固定されています：

- L5の閾値（`LEADER_THRESHOLD`, `ON/OFF_THRESHOLD`, `EMA_ALPHA`）
- L4の活性化カウンタ増加ルール
- 各層の正規化定数
- クオリアの反応設定（danger→fear上昇、goal→joy上昇など）

これらは「このコードの生まれながらの値」であり、論文§2.3.2の「DNA初期値＋学習値」の枠組みに従います。「正しい値」ではなく「出発点」です。

**コミュニティ検証に開かれた研究対象**:
- 各層の活動量スケールの最適化
- 環境を変えても安定する閾値の探索
- 「人間っぽい」挙動を生むパラメータ範囲の特定

v1.1で「環境を超えて意識ON率約70%で安定」という観察があったように、v2.0でも環境を超えて安定するパラメータ領域があるかもしれません。

## v1.1からv2.0への変更点

| 項目 | v1.1 | v2.0 |
|---|---|---|
| 同期の構造 | 4層が対等に活動 | 主導層と追随層の非対称関係 |
| 計算式 | `0.25 × 4層の合算` | `主導強度 × 追随整合性` |
| 意識の記述 | 同期スコアのみ | `sync_score` + `sync_type` |
| 活動量の定義 | 状態の絶対値 | 状態の変化量 |
| L1の役割 | センサー | センサー＋同期信号源＋現実接地 |
| 説明範囲 | 通常の覚醒意識 | 覚醒・夢・フロー・瞑想・各種状態 |

v2.0はv1.1を否定するものではなく、より一般的な枠組みへの拡張です。v1.1の対称的同期は、v2.0の理論上は「複数層の活動量が近く、明確な主導層が前景化しない均衡的同期状態」として位置づけられます。ただし、現行の簡易実装では均衡的同期状態は明示的には実装しておらず、主導層なしは `sync_type=None` として扱います。

詳細は[論文 Changelog](HIDA_theory_paper_v2_0.md) 参照。

## 理論的背景

### 関連する既存理論

**予測符号化（Predictive Coding, Friston）**
- 脳が常に予測を行い、予測誤差を最小化するという理論
- 本システムの第3層がこの機構を実装
- v2.0ではL3の予測誤差を「L3が同期信号を発するための活動量」として位置づける

**統合情報理論（IIT, Tononi）**
- 意識をΦとして数値化
- v2.0のsync_scoreはΦと類似する「同期強度」を測るが、加えて「同期のタイプ（sync_type）」を導入する点が異なる

**グローバルワークスペース理論（GWT, Baars/Dehaene）**
- 情報の広域共有として意識を説明
- v2.0の「追随整合性」が広域共有の現象と対応

**Anthropic 171 emotion vectors**
- LLM内部で発見された機能的感情ベクトル
- v2.0のL2構造と独立に発見された類似構造（収束的証拠）
