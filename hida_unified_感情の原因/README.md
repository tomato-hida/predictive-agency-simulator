# HIDA - Human-Inspired Dynamic Architecture

**5層意識モデルの実装プロトタイプ**

> 「意識とは、4層が同期した状態である」

※ L5は「追加の層」ではなく、L1〜L4の同期状態として表現している。5層と呼ぶのは便宜上の名称。

---

## 概要

HIDAは「人の内なる運動理論」をベースにした意識モデルの実装です。  
L1〜L4の処理層が同期したとき、L5（意識）が成立するという設計になっています。

LLMは「言語化レイヤ」として隔離されており、未接続でも動作します。  
主役はL1〜L5のループです。

---

## 5層構造

```
L1: 身体層       ── 環境との接点、エネルギー管理
L2: クオリア層   ── 感情的評価（valence/arousal + 個別感情）
L3: 予測層       ── 予測誤差の検出
L4: 記憶層       ── 内部マップ、長期記憶、変調値（性格形成）
─────────────────────────────────────────
L5: 意識層       ── L1〜L4が同期した状態（実体ではなく状態）
```

**情報の流れ：**
```
環境 → L1 → L2 → L3 → L4 → 行動 → 環境
             ↑___________________________|
                    閉じたループ
```

## 設計上の特徴

### 感情＋原因の分離
感情（例：怒り）と「なぜそう感じたか」を別々に記録し、それぞれ異なる速度で忘れる。

```python
# NPCに邪魔された
self.l2.qualia['anger'] += 0.3
self.l2.set_cause('anger', "ボブに進路を塞がれた")
```

怒りは長く残るが、原因は先に忘れる（anger_decayの設定による）。

### 点滅意識
各層の活動度を0〜1に正規化し、EMA（指数移動平均）で平滑化することで「意識のON/OFF」を実現。  
平和な移動中は意識がOFF、新発見やトラブル時にONになる。

```
予測誤差が大きい → 同期トリガー → 意識ON → 行動モード切替（注意/探索/抑制）
予測誤差が消える → 同期解除  → 意識OFF → 自動運転
```

LLMへの言語化は意識ONのタイミングで行われる任意の副作用であり、主役ではない。

### 経験による性格形成
長期記憶（LTM）から変調値（`fear_weight`, `safe_preference`）が更新され、セッションをまたいで性格が変わる。  
危険な経験を繰り返すと臆病になり、時間が経つと薄れる。

## 動作環境

- Python 3.8+
- 外部ライブラリ不要（標準ライブラリのみ）
- LLM連携（オプション）：`ollama`（ローカル）または Anthropic API

## 使い方

```bash
# 通常実行
python hida_unified.py

# 低エネルギーで5回繰り返し（臆病化の観察）
python hida_unified.py loop

# 高エネルギーで5回繰り返し（冒険的な性格の観察）
python hida_unified.py loop_high

# 記憶・変調値のリセット
python hida_unified.py reset
```

## LLM連携

同期している（意識ON）タイミングで、状態をパッケージ化してLLMに渡す。

```
【感情状態】
- 怒り: 80%
- 怒りの原因: ボブに進路を塞がれた   ← 覚えていれば
- 恐怖: 30%
- 恐怖の原因: 不明                   ← 忘れていたら不明
```

LLMはこれを受け取って1人称で言語化する。  
未接続の場合は `(LLM未接続)` とフォールバックし、ループは継続する。
